## 页面1: 总览

### 1.1 页面描述

展示配置中心的整体运行状态和实施进度，包括资源统计、核心指标、阶段计划等内容。

### 1.2 UI结构

```
┌─────────────────────────────────────────────────────────┐
│ 页面标题: 配置中心总览                                 │
│ 副标题: 当前工作台按plan基线展示实施进度...                │
├─────────────────────────────────────────────────────────┤
│ 统计卡片组 (6个)                                      │
│ ┌─────────┬─────────┬─────────┬─────────┐           │
│ │ 页面资源 │ 生效提示 │ 生效作业 │ 生效API │           │
│ │ 数量    │ 规则数  │ 场景数  │ 注册数  │           │
│ └─────────┴─────────┴─────────┴─────────┘           │
│ ┌─────────┬─────────┐                               │
│ │ 角色总数 │ 待处理   │                               │
│ │         │ 事项     │                               │
│ └─────────┴─────────┘                               │
├─────────────────────────────────────────────────────────┤
│ 核心指标 + 阶段计划 + 当前基线                         │
│ ┌─────────┬─────────┬─────────┐                     │
│ │ 核心指标 │ 阶段计划 │ 当前基线 │                     │
│ │(4个子项)│(6个阶段)│(List)   │                     │
│ └─────────┴─────────┴─────────┘                     │
└─────────────────────────────────────────────────────────┘
```

### 1.3 组件规格

#### 统计卡片
- 标题: 14px, 次要文字颜色
- 数值: 30px, 主要文字颜色, 600 weight
- 交互: hover时边框高亮
- Loading: 支持骨架屏

#### 核心指标卡片
- 包含4个子指标:
  - 作业成功率 (百分比)
  - 平均节省时长 (秒)
  - 已失效对象 (数量)
  - 即将到期 (数量)

#### 阶段计划卡片
- List列表, 6项
- 每项格式: "Phase X: 详细说明"
- 字体: 14px

#### 当前基线卡片
- List列表, 5条原则
- 底部显示Tag组: WP-A ~ WP-F + Mock Data + Admin Web P0

## 页面2: 页面资源中心

### 2.1 页面描述

统一维护页面识别规则与元素映射，支持站点/菜单筛选，可管理页面资源和元素映射。

### 2.2 UI结构

```
┌─────────────────────────────────────────────────────────┐
│ 页面标题: 页面资源中心                                 │
│ 副标题: 统一维护页面识别规则与元素映射...                    │
├─────────────────────────────────────────────────────────┤
│ 统计卡片: 站点数 | 菜单数 | 页面资源数                  │
├─────────────────────────────────────────────────────────┤
│ 筛选栏: 站点(Segmented) + 菜单(Select) + 新建按钮    │
├─────────────────────────────────────────────────────────┤
│ 表格: 页面资源列表                                      │
│ 列: 名称 | 所属菜单 | 版本 | 元素数 | 识别口径 | 状态 │
│ 操作列: 编辑 | 元素映射                                 │
├─────────────────────────────────────────────────────────┤
│ Modal: 新建/编辑页面资源                                 │
│ 表单字段:                                              │
│ - ID (只读)                                          │
│ - 名称 (必填, 最大128字符)                             │
│ - 菜单 (下拉选择)                                      │
│ - 状态 (下拉: DRAFT/ACTIVE/DISABLED/EXPIRED)         │
│ - 组织范围 (必填)                                      │
│ - 当前版本 (数字, 最小1)                                │
│ - 元素数量 (只读)                                      │
│ - 识别口径 (必填)                                      │
├─────────────────────────────────────────────────────────┤
│ Drawer: 元素映射管理                                    │
│ 标题: "元素映射: [页面资源名称]"                           │
│ 表格: 元素映射列表                                      │
│ 列: 逻辑名 | 选择器(代码样式) | 类型 | 必填 | 更新时间   │
│ 操作列: 编辑 | 删除(带确认)                            │
│ 新增按钮: Drawer右上角                                 │
├─────────────────────────────────────────────────────────┤
│ Modal: 新建/编辑元素映射                                 │
│ 表单字段:                                              │
│ - ID (只读)                                          │
│ - 页面资源ID (只读)                                   │
│ - 逻辑名 (必填)                                       │
│ - 定位表达式 (必填)                                    │
│ - 选择器类型 (下拉: XPATH/CSS)                        │
│ - 是否必填 (Switch: 是/否)                             │
└─────────────────────────────────────────────────────────┘
```

### 2.3 筛选逻辑

- 站点筛选: 点击自动过滤菜单选项
- 菜单筛选: 根据站点和菜单筛选页面资源
- 重置逻辑: 切换站点时自动重置菜单为"全部"

### 2.4 操作规范

- 删除元素: 需要确认对话框
- 状态切换: 需要二次确认
- 未保存: 关闭Modal/Drawer前检测

### 3.3 字段规范

#### 输入参数字段
- 参数名: Input, 必填
- 描述: Input
- 类型: Select (STRING/NUMBER/BOOLEAN/OBJECT/ARRAY)
- 来源: Select (固定值/页面元素/API输出/上下文)
- 值/映射: Input, placeholder: 如 customer_id / $.data.score
- 必填: Switch

#### 输出参数字段
- 字段名: Input, 必填
- 路径: Input, 格式: $.data.字段名
- 描述: Input
- 类型: Select (同输入参数)
- 属性: 按钮 (仅OBJECT/ARRAY类型显示)

- 测试环境: 蓝色Tag (#1890ff)

### 5.3 条件构建器规范

#### 规则组
- 逻辑: AND (所有条件满足) / OR (任一条件满足)
- 嵌套: 支持最多3层嵌套

#### 规则项
- 字段来源: 页面元素 / 固定值 / API输出 / 上下文
- 字段: 根据来源动态显示字段选择
- 操作符: 等于 / 不等于 / 包含 / 不包含 / 大于 / 小于 / 为空 / 不为空
- 值: 根据字段类型显示相应输入框

#### 提示模式
- 模式类型:
  - 强制弹窗: 强制显示，必须确认
  - 智能提示: 智能判断是否显示
  - 悬浮按钮: 不弹窗，显示悬浮按钮

#### 关闭模式
- 模式类型:
  - 立即关闭: 确认后立即关闭
  - 延时关闭: 确认后N秒自动关闭
  - 手动关闭: 需要用户手动关闭

## 🔄 交互流程规范

### 1. CRUD操作流程

```
新建流程:
1. 点击"新建"按钮
2. 打开Modal/Drawer，显示空表单
3. 自动生成ID（只读显示）
4. 用户填写表单
5. 点击"保存"
6. 表单验证
7. 调用API创建
8. 显示成功消息
9. 关闭Modal/Drawer
10. 刷新列表

编辑流程:
1. 点击"编辑"按钮
2. 打开Modal/Drawer，填充数据
3. 用户修改表单
4. 点击"保存"
5. 表单验证
6. 调用API更新
7. 显示成功消息
8. 关闭Modal/Drawer
9. 刷新列表

删除流程:
1. 点击"删除"按钮
2. 显示确认对话框
3. 用户确认
4. 调用API删除
5. 显示成功消息
6. 刷新列表

状态切换流程:
1. 点击"启用/停用"按钮
2. 显示确认对话框
3. 用户确认
4. 调用API切换状态
5. 显示成功消息
6. 刷新列表
```

### 2. 表单验证流程

```
表单验证:
1. 实时验证: 输入时即时验证
2. 提交验证: 点击保存时完整验证
3. 显示错误: 错误字段下方显示错误消息
4. 阻止提交: 有错误时阻止表单提交

必填验证:
- 字段为空时显示"请输入[字段名]"
- 下拉为空时显示"请选择[字段名]"

格式验证:
- URL格式: "请输入有效的URL"
- JSON格式: "JSON格式错误，请检查"
- 数字范围: "请输入X-Y之间的数字"

长度验证:
- 超过最大长度: "[字段名]不能超过X字符"
```

### 3. 未保存检测流程

```
未保存检测:
1. 表单打开时记录初始值
2. 用户修改表单时设置hasUnsaved=true
3. 用户点击关闭/取消时:
   a. 如果hasUnsaved=true:
      i. 显示确认对话框
      ii. 用户选择"离开" → 允许关闭
      iii. 用户选择"继续编辑" → 阻止关闭
   b. 如果hasunseaved=false:
      i. 直接关闭

页面刷新检测:
- beforeunload事件: hasUnsaved=true时阻止刷新
```

### 4. 筛选交互流程

```
筛选交互:
1. 用户选择筛选条件
2. 列表即时更新
3. 更新URL参数（可选）

级联筛选:
1. 主筛选改变
2. 重置子筛选为"全部"
3. 更新子筛选选项
4. 刷新列表

重置筛选:
1. 点击"全部"
2. 重置所有筛选条件
3. 显示全部数据
```

### 5. 分页交互流程

```
分页交互:
1. 显示当前页/总页数
2. 上一页/下一页按钮
3. 跳转到指定页
4. 改变每页显示数

分页状态:
- 当前页码
- 每页数量
- 总记录数
- 总页数

加载状态:
- 切换页时显示Loading
- 数据加载完成后隐藏
```

### 6. 异常处理流程

```
API调用异常:
1. 捕获错误
2. 显示错误消息
3. 按错误类型显示不同提示:
   - 网络错误: "网络连接失败，请检查网络"
   - 服务器错误: "服务器错误，请稍后重试"
   - 业务错误: 显示具体错误消息

超时处理:
1. 设置超时时间
2. 超时后显示提示
3. 允许用户重试

加载失败:
1. 显示错误提示
2. 提供"重试"按钮
3. 点击后重新加载数据
```

### 7. 权限控制流程

```
权限检查:
1. 用户登录时获取角色和权限
2. 根据权限显示/隐藏功能
3. 无权限时禁用按钮或显示提示

按钮权限:
- 有权限: 正常显示，可点击
- 无权限: 禁用或隐藏，悬停显示"无权限"

页面权限:
- 有权限: 正常显示
- 无权限: 跳转首页或显示403页面

数据权限:
- 有权限: 显示数据
- 无权限: 显示"无权限"提示
```

### 8. 调试流程

```
API调试流程:
1. 选择调试环境 (测试/生产)
2. 显示当前请求路径
3. 输入请求JSON
4. 点击"执行调试"
5. 显示Loading状态
6. 模拟返回响应
7. 显示请求预览和响应预览

节点测试流程:
1. 选择节点
2. 配置节点参数
3. 点击"测试节点"
4. 显示测试结果
5. 验证节点配置

作业预览流程:
1. 打开预览确认Modal
2. 加载预览数据
3. 显示字段列表
4. 用户勾选要写入的字段
5. (异常字段禁用勾选)
6. 点击"确认写入"
7. 执行作业
```
