# 配置中心研发防漏清单

> 适用范围：`config-center` 前端与 mock/service 联动改造  
> 对应决策：`spec/config-center-product/decision-ledger.md` / `D-023`

## 1. 触发条件
以下任一情况命中时，必须执行本清单：

1. 新增或修改取值来源（页面元素、API、上下文、固定值）。
2. 新增或修改字段映射、入参映射、出参映射。
3. 新增或修改条件编辑、API注册、作业节点任一配置能力。

## 2. 五项联动检查（必须全通过）
1. 对象模型：`src/types.ts` 与存储结构已补齐。
2. 配置 UI：所有消费模块都可配置，不允许只改单页面。
3. 保存结构：提交 payload、回显结构、mock/service 解析一致。
4. 运行解析：预览/执行逻辑能识别新字段，异常可提示可阻断。
5. 验收用例：覆盖“可配置、可保存、可回显、可执行、异常阻断”。

## 3. 提交前门禁（本地自检）
1. 关键字段全局检索：
```powershell
rg -n "<关键字段名>" src spec
```
2. 构建校验：
```powershell
npm run build
```
3. PR 描述必须包含“联动影响面清单”：
- 模块：例如 `API注册 / 智能提示 / 智能作业 / 服务层`
- 文件：列出实际改动文件
- 校验结果：`rg` 检查结果 + `npm run build` 结果

## 4. 本次问题复盘（案例）
问题：API注册支持页面元素取值，但条件编辑缺少设置值流程。  
根因：配置能力跨模块联动不完整（UI 与数据链路未同时打通）。  
防线：后续一律按本清单执行，未完成五项联动不得合并。
