# 营小助配置中心需求规格说明（Spec）
> 状态：已确认  
> 创建日期：2026-03-11  
> 最后更新：2026-03-11  
> 所属里程碑：MVP（V1.0）

## 0. 评审摘要

### 0.1 一句话版本
营小助配置中心是面向银行内部运营与审核人员的浏览器端辅助能力平台，通过 JSSDK 在业务页面提供“智能提示 + 智能作业”，提升作业准确率与处理效率。

### 0.2 改前 / 改后
改前：监管要求传达、风险提醒和作业指引分散在公告、群消息与人工经验中；内部系统数据需要人工查询并手工录入业务系统页面。  
改后：业务系统预注入营小助 JSSDK，页面可实时展示监管与风险提示，并自动获取内部系统数据后按规则注入当前页面字段。

### 0.3 典型场景
审核人员办理对公开户时，页面自动弹出该客户相关监管临时要求和风险提醒；同时从内部核验系统自动拉取企业信息并填入当前表单，人员仅需复核与确认提交。

### 0.4 本次不做
1. 不改造各业务系统核心流程与页面主架构。
2. 不建设独立桌面客户端（仅浏览器环境）。
3. 不覆盖审批流引擎重构（只对接现有审批结果或页面流程）。

### 0.5 待拍板事项
1. 无。

## 1. 概述

### 1.1 功能简介
营小助配置中心提供两块核心能力：
1. 智能提示：监管临时要求传达、事中风险提醒、新员工作业指引。
2. 智能作业：从内部系统获取数据并自动注入当前业务页面。
3. 能力联动：智能作业触发完全由智能提示控制，提示命中后可按配置执行智能作业。
4. 预览模式：智能作业支持“填充前预览并可编辑”后再确认注入。
5. 悬浮触发：智能提示命中后可按配置在页面注入悬浮按钮，便于重复触发对应智能作业。
6. 场景编排：智能作业支持场景编排，编排由组件节点串联执行，首批节点包含页面取值、页面赋值、接口调用、JavaScript 脚本执行，并支持节点能力扩展。

两类能力以 JSSDK 形式运行在浏览器环境，由业务系统提前注入到网页中。

### 1.2 背景与目标
- 背景：银行一线运营与审核场景规则变化快、合规要求高、页面录入项多，人工作业依赖经验，容易出现漏看、漏填、误填。
- 目标：
1. 将关键监管要求触达时效提升至分钟级。
2. 将人工录入与查数步骤减少 30% 以上。
3. 将因遗漏风险提示导致的差错率降低 20% 以上。

### 1.3 所属模块
1. 规则与内容配置模块（提示规则、指引内容、注入映射配置）
2. 接口配置子项模块（接口定义、参数映射、鉴权与发布）
3. 运行时决策模块（页面上下文识别、触发判断、优先级排序）
4. 智能作业编排模块（编排定义、节点执行、上下文传递）
5. JSSDK 执行模块（提示渲染、数据注入、事件回传）
6. 审计与运营模块（命中统计、效果分析、日志追踪）

## 2. 用户故事与场景

### 2.1 目标用户
1. 银行运营人员（柜面/后台运营）
2. 银行审核人员（授信、开户、交易审核等岗位）
3. 规则运营人员（维护提示内容与触发规则）
4. 管理与合规人员（监控执行效果与审计追踪）

### 2.2 用户故事
1. 作为审核人员，我希望在办理业务时自动看到当前场景相关的监管临时要求，避免遗漏关键合规动作。
2. 作为运营人员，我希望系统在录入页面自动补全内部系统权威数据，减少重复查询与手工输入。
3. 作为新员工，我希望系统按当前步骤给出作业指引，降低学习成本与误操作率。
4. 作为规则运营，我希望不改代码即可更新提示与注入规则，并快速生效。

### 2.3 使用场景
1. 监管临时通知发布后，对涉及业务页面实时弹出提示并附执行要点。
2. 审核关键字段录入时，命中风险规则并给出分级提醒与处理建议。
3. 新员工作业时按页面步骤显示操作指引、示例与常见错误。
4. 客户资料录入时自动从内部系统数据源拉取信息并注入当前表单。
5. 客户资料录入时先展示待填充预览，用户可修改后确认写入页面。
6. 对需要反复执行的作业，提示命中后可注入悬浮按钮，用户点击可重复触发该提示绑定的智能作业。
7. 对复杂作业场景，提示命中后触发场景编排，系统按节点顺序执行页面取值、接口调用、脚本处理与页面赋值等步骤。

### 2.4 用户旅程（已确认）
1. 一线办理旅程
- 用户进入已启用菜单页面后，外部 SDK 监听 DOM 变化并通知智能提示 SDK。
- 智能提示 SDK 按条件表达式取值（固定值/页面元素/接口值）执行 OR/AND 规则，命中提示后弹窗展示。
- 多条提示同轮命中时，合并到同一提示窗口展示。
- 若提示规则配置了联动作业，则在提示渲染后按触发顺序执行智能作业。
- 若联动作业依赖接口参数，系统在调用接口前按映射规则从页面元素提取参数值并组装请求。
- 若联动作业启用场景编排，系统按编排节点顺序执行（页面取值 -> 接口调用/脚本处理 -> 页面赋值等），节点结果写入共享上下文供后续节点引用。
- 智能作业若启用预览模式，则在编排全部计算完成后统一展示待填充数据，用户可修改并确认后一次性注入。
- 若该提示规则配置了悬浮按钮，用户可通过悬浮按钮重复触发该提示绑定的智能作业。
- 智能作业完成注入后不再进行后续处理；流程保持不变。

2. 管理员配置旅程
- 管理员先在“接口配置子项”中完成接口定义、参数来源映射（固定值/页面元素）与发布，再在业务菜单级配置启停、提示规则、联动作业与生效策略。
- 规则发布后无需业务系统改代码即可生效。
- 通过运行数据持续优化规则质量与命中效果。

## 3. 功能需求

### 3.1 输入
1. 页面上下文：系统标识、页面标识、业务类型、当前字段、用户角色。
2. 规则配置：触发条件、提示内容、风险等级、注入字段映射、优先级。
3. 智能提示条件输入：条件表达式（左值、比较运算符、右值）。
4. 条件取值来源配置：左值与右值均支持固定值、页面元素（XPath）、接口返回值。
5. 接口子项配置：接口标识、接口地址/方法、入参与页面字段映射（支持页面取值）、鉴权策略、发布状态。
6. 内部系统数据查询入参：客户标识、证件号、企业统一社会信用代码等。
7. 接入配置：JSSDK 初始化参数、鉴权参数、日志回传开关。
8. 联动动作配置：提示规则命中后是否执行智能作业、执行的作业模板/映射规则标识。
9. 智能作业预览配置：是否启用预览模式、可编辑字段范围、确认/取消行为配置、预览时机（编排场景固定为“编排完成后统一预览”）。
10. 悬浮按钮配置：提示命中后是否注入、按钮样式与位置、绑定作业标识、可见条件与权限范围。
11. 智能提示条件项配置：`left_preprocessors`、`right_preprocessors`（左右值预处理器链）。
12. 参数映射项配置：`param_name`、`source_type`、`source_value`、`required`、`value_type`、`default_value`、`test_value`。
13. 场景编排配置：`orchestration_id`、节点清单、节点顺序、节点输入映射、节点输出映射、启停状态（失败策略平台固定为 `STOP`，前端不开放配置）。

### 3.2 输出
1. 智能提示输出：提示文案、风险等级、操作建议、来源与生效时间。
2. 智能作业输出：注入字段值、来源标识、置信度（如有）、注入状态。
3. 运行日志输出：触发事件、用户操作、注入结果、异常原因。
4. 运营统计输出：提示命中率、采纳率、注入成功率、平均节省时长。
5. 联动执行输出：提示命中后是否触发作业、联动作业执行结果与失败原因。
6. 预览输出：待填充字段清单、用户修改值、确认/取消结果。
7. 触发来源输出：提示命中触发/提示控制下悬浮按钮触发。
8. 条件运算日志输出：左右值预处理执行结果、条件判定结果、命中与未命中原因。
9. 参数映射日志输出：参数取值来源、参数组装结果、调用跳过原因。
10. 编排执行输出：编排实例状态、节点执行顺序、节点输入输出快照、失败节点与失败原因。

### 3.3 核心行为描述
1. 智能提示能力
- 监管临时要求传达：根据业务类型、IP、人员与机构混合策略定向触达。
- 事中风险提醒：在关键节点触发分级提醒（高/中/低）。
- 新员工作业指引：按页面步骤与字段上下文给出操作指引，新员工身份由人工标记。
- 触发机制：每条提示由一组条件表达式组成，单条件表达式为“左值 + 运算符 + 右值”；条件关系支持“任意满足（OR）”和“全部满足（AND）”。
- 取值机制：条件表达式左值与右值均支持固定值、页面元素（XPath）与接口返回值，可混合组合。
- 预处理机制：条件表达式左值与右值均支持独立预处理器链，按配置顺序执行后再参与运算；首批内置“提取前几位”“数字转换”。
- 触发与取值机制：由外部 SDK 监听 DOM 变化并通知智能提示 SDK；智能提示 SDK 收到通知后完成页面取值与接口取值并执行规则运算，条件满足则弹出提示。
- 联动机制：提示规则命中后可按规则配置选择执行智能作业。
- 接口治理机制：接口值来源必须引用“接口配置子项”中已定义并已发布的接口。
- 参数组装机制：当接口参数来源配置为页面元素时，调用前按映射规则从页面取值并组装接口请求。

2. 智能作业能力
- 从内部系统数据源发起查询并做结果标准化。
- 按注入映射规则写入当前页面字段。
- 注入冲突时遵循策略：保留人工值、覆盖旧值或弹窗确认（策略可配置）。
- 触发方式仅支持智能提示控制链路：由智能提示规则命中后触发。
- 联动触发场景下，多条作业按智能提示触发顺序执行。
- 预览模式：启用后在编排全部计算完成后统一展示待填充数据，允许用户修改；用户确认后才执行一次性注入。
- 悬浮按钮触发：仅在提示命中后按配置注入悬浮按钮，用户点击可重复触发该提示绑定作业。
- 场景编排：智能作业可配置为节点化编排流程，按顺序执行并在节点间传递上下文数据。
- 绑定关系：单个智能作业绑定一个场景编排定义，触发后执行该编排全链路。
- 首批内置节点：获取页面元素值、页面字段赋值、接口调用、执行 JavaScript 脚本。
- 脚本节点能力：当前不限制脚本节点直接读写页面对象，亦可读写编排上下文。
- 节点扩展：支持新增自定义节点类型，扩展后可在编排中被引用。

3. JSSDK 运行机制
- 业务系统在页面预先注入 JSSDK。
- JSSDK 在浏览器环境采集上下文并请求决策服务。
- 返回结果后进行提示渲染与字段注入，并回传埋点日志。
- 智能作业可在提示命中后按配置注入悬浮按钮，点击事件进入提示控制的统一作业触发链路并回传日志。

4. 配置生效机制
- 规则配置支持版本化与灰度发布（按机构/系统/角色）。
- 规则更新无需业务系统重新发布代码。
- 智能提示默认采用 IP + 人员 + 机构混合生效策略。

### 3.4 业务规则与约束
1. 同一页面同一字段同一时刻只执行一条注入动作。
2. 智能作业触发完全由智能提示控制，不提供独立触发入口。
3. 高风险提示在用户确认处理前不可静默消失。
4. 所有自动注入必须可追溯来源、时间和规则版本。
5. 涉及客户敏感信息展示时必须按脱敏策略处理。
6. JSSDK 仅在已注入页面生效，不主动侵入未接入系统。
7. 新员工标记采用人工维护，不改变现有业务流程。
8. 智能提示规则支持条件组合关系：任意满足（OR）与全部满足（AND）。
9. 条件表达式左值与右值来源支持固定值、页面元素（XPath）、接口返回值，可任意组合。
10. 条件取值失败（页面元素不存在/值为空/格式非法/接口调用失败或超时）时，该条件按“不满足”处理并记录运行日志。
11. 页面元素定位方式统一为 XPath。
12. 条件运算符需支持常见运算符集合（等于、不等于、大于、小于、大于等于、小于等于、包含、不包含、in、not in、正则匹配、为空、非空）。
13. 每条智能提示规则可配置“是否联动智能作业”，开启后提示命中即触发指定智能作业。
14. 联动执行失败不阻断现有业务流程，但必须记录失败原因并可追溯。
15. 多条提示同轮命中时，须合并到同一提示窗口展示。
16. 联动作业按智能提示触发顺序执行，不按优先级抢占。
17. DOM 高频变化场景需执行防抖，避免重复弹窗与重复联动作业。
18. 智能作业注入完成后不再进行后续处理；仅当用户点击提示控制下的悬浮按钮时允许再次触发。
19. 智能作业支持可配置预览模式；启用预览时，必须在用户确认后才能写入页面。
20. 预览模式下用户取消填充时，不执行注入并记录取消原因（可为空）。
21. 悬浮按钮仅对已授权用户和已启用菜单可见，且仅在对应提示命中后展示。
22. 悬浮按钮重复触发需受防抖与频控约束，避免短时间重复注入。
23. 接口作为配置管理子项独立维护；未定义或未发布接口不得被提示条件表达式引用。
24. 接口标识（`interface_id`）在同一租户/项目范围内必须唯一。
25. 接口配置状态仅允许 `DRAFT`、`PUBLISHED`、`OFFLINE`；仅 `PUBLISHED` 状态可被规则引用。
26. 接口版本策略当前不启用；保留 `version` 字段作为后续扩展位，不参与当前规则引用与发布流程。
27. 接口鉴权密钥类配置必须加密存储，页面与日志禁止明文展示。
28. 接口下线受引用保护：被已发布规则引用的接口不允许直接下线，需先解除引用。
29. 接口调用参数 `timeout_ms`、`retry_count` 由平台统一控制，不对用户开放配置。
30. 接口发布与下线权限仅限接口 `owner` 与平台管理员。
31. 若接口定义请求中携带 `timeout_ms` 或 `retry_count` 用户自定义值，系统必须忽略并使用平台统一值。
32. 前端接口配置页面不得展示 `timeout_ms`、`retry_count` 的可配置项。
33. 接口参数支持“固定值 + 页面元素取值”混合组装；若必填页面参数缺失则本次接口调用跳过并记录原因。
34. 参数映射项支持 `test_value`，用于联调与规则验证阶段的参数模拟取值。
35. 参数映射值默认按字符串处理；未显式指定 `value_type` 时按 `string` 处理。
36. 智能提示条件表达式左值与右值均支持可扩展预处理器链，且左右链路独立配置；首批预处理器为“提取前几位”“数字转换”。
37. 条件预处理失败视为该条件不满足，并记录失败原因。
38. 参数与调用日志采用全量落库策略；查询与展示时按敏感级别脱敏。
39. 预处理器能力后续由系统管理员在配置中心随时扩展（新增/启停），不影响既有规则运行。
40. 预处理器统一使用 JavaScript 实现，必须通过平台沙箱执行。
41. JS 预处理器执行超时或异常时，当前条件按不满足处理，且不影响其他条件继续计算。
42. 首批预处理器仅内置“提取前几位”“数字转换”两项。
43. 预处理器扩展仅允许系统管理员发布与启停，普通用户不可直接管理。
44. 智能作业支持场景编排；单个智能作业绑定一个编排定义，执行入口仍仅能由智能提示触发。
45. 编排首批节点类型至少包含：页面取值节点、页面赋值节点、接口调用节点、JavaScript 脚本节点。
46. 编排默认按节点顺序串行执行；同一编排实例内不并行执行节点。
47. 节点可读取前序节点输出作为输入；若必需输入缺失，当前节点执行失败并记录原因。
48. 编排节点执行失败时，当前且唯一策略为终止当前编排实例（`STOP`）；前端不开放“继续执行”配置；失败不阻断业务主流程但必须留痕。
49. JavaScript 脚本节点当前允许按配置直接读写页面对象与上下文数据，执行行为须全量留痕。
50. 自定义节点扩展仅允许系统管理员发布/启停；发布前需完成自测并通过；停用后新编排不可再引用该节点类型。
51. 悬浮按钮每次点击均创建新的编排实例，不复用上一次实例上下文。
52. 编排场景启用预览模式时，预览窗口在“编排全部计算完成后”统一弹出，确认后一次性写入页面。

### 3.5 接口子项配置模型
接口子项用于管理“规则可引用的接口来源值”，需先定义并发布，再被提示条件表达式引用。

1. 字段清单（建议）
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `interface_id` | string | 是 | 接口唯一标识，规则引用主键 |
| `interface_name` | string | 是 | 接口名称，便于运营识别 |
| `domain` | string | 是 | 接口域名或服务标识 |
| `path` | string | 是 | 接口路径 |
| `method` | enum | 是 | `GET`/`POST`/`PUT`/`DELETE` |
| `headers_template` | object | 否 | 请求头模板，支持变量占位 |
| `query_mapping` | object | 否 | URL 查询参数映射，支持固定值与页面元素取值表达式 |
| `body_mapping` | object | 否 | 请求体参数映射，支持固定值与页面元素取值表达式 |
| `auth_type` | enum | 是 | `NONE`/`TOKEN`/`AKSK`/`CUSTOM` |
| `auth_config` | object | 否 | 鉴权配置（敏感字段加密） |
| `timeout_ms` | int | 否（平台内置） | 超时阈值，平台统一控制（默认 1000），前端不展示 |
| `retry_count` | int | 否（平台内置） | 重试次数，平台统一控制（默认 1），前端不展示 |
| `response_path` | string | 是 | 从响应中取值路径（如 JSONPath） |
| `value_type` | enum | 是 | `string`/`number`/`boolean`/`array`/`object` |
| `status` | enum | 是 | `DRAFT`/`PUBLISHED`/`OFFLINE` |
| `version` | int | 否 | 预留扩展字段，当前版本策略不启用 |
| `owner` | string | 是 | 责任人账号 |
| `updated_at` | datetime | 是 | 更新时间 |

2. 状态流转
- `DRAFT -> PUBLISHED`：完成校验并发布后可被规则引用。
- `PUBLISHED -> OFFLINE`：仅在无已发布规则引用时允许下线。
- 版本策略当前不启用，后续可扩展为多版本流转模型。

3. 校验规则
- 发布前必须校验：接口连通性、鉴权有效性、`response_path` 可解析性。
- 引用校验必须校验：接口状态为 `PUBLISHED` 且接口标识存在。
- 安全校验必须校验：敏感字段（token、secret）不允许明文回显。
- 权限校验必须校验：仅 `owner` 与平台管理员可执行发布/下线。
- 参数校验必须校验：配置端不接受用户自定义 `timeout_ms`、`retry_count`。
- 映射校验必须校验：页面取值映射表达式格式合法（XPath 可解析），缺失必填页面参数时运行期跳过调用并留痕。

### 3.6 页面参数映射规范
用于定义“接口调用前如何组装参数”，支持固定值与页面取值混合模式。

1. 映射项结构
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `param_name` | string | 是 | 目标参数名 |
| `source_type` | enum | 是 | `fixed` / `page` |
| `source_value` | string | 是 | 固定值或页面 XPath 表达式 |
| `required` | boolean | 是 | 是否必填参数 |
| `value_type` | enum | 否 | 参数类型，默认 `string` |
| `default_value` | string | 否 | 非必填参数取值失败时的默认值 |
| `test_value` | string | 否 | 测试/联调阶段模拟值 |

2. 取值与组装规则
- `source_type=fixed`：直接使用 `source_value`。
- `source_type=page`：按 XPath 从页面取值。
- `required=true` 且取值失败：跳过本次调用并记录失败原因。
- `required=false` 且取值失败：优先用 `default_value`，无默认则不传该参数。
- `value_type` 未配置时默认按字符串处理。
- `test_value` 在测试模式下优先生效，不影响生产模式取值。
- 接口参数映射层不支持预处理器链，保持参数组装逻辑简化。

3. 日志与展示规则
- 参数取值、组装与调用结果全量落库；条件左右值预处理执行日志同样全量落库。
- 日志查询与页面展示默认脱敏（按参数敏感级别与角色权限控制）。

### 3.7 JS 预处理器规范
用于定义智能提示条件表达式左右值的预处理执行标准。

1. 执行模型
- 执行语言：JavaScript。
- 执行位置：平台受控沙箱运行时（与业务页面隔离）。
- 调用时机：左右值取值完成后，按 `left_preprocessors`、`right_preprocessors` 顺序分别执行。
- 执行结果：输出处理后值，参与后续条件运算。

2. 函数签名（规范）
```javascript
function transform(input, options, context) {
  // input: 原始值
  // options: 预处理器参数
  // context: 执行上下文（规则ID、表达式ID、字段元信息等只读数据）
  return output;
}
```

3. 沙箱与安全约束
- 禁止访问页面对象与宿主环境敏感对象（如 `window`、`document`、`localStorage`）。
- 禁止网络访问、动态代码执行（如 `eval`、`Function`）。
- 仅允许白名单内标准库能力。
- 单次执行必须受超时限制（建议 <= 50ms）。

4. 首批内置预处理器
| 标识 | 名称 | 参数 | 输入示例 | 输出示例 |
| --- | --- | --- | --- | --- |
| `prefix_extract` | 提取前几位 | `length`(int, >0) | input=`"6222021234567890"`, length=6 | `"622202"` |
| `to_number` | 数字转换 | 无 | input=`"00123.40"` | `123.4` |

5. 扩展机制（管理员）
- 系统管理员可新增自定义 JS 预处理器并配置：标识、名称、参数 Schema、JS 实现代码、启停状态。
- 新增后需通过测试值验证与安全检查，发布成功后方可被规则引用。
- 启停不影响历史日志查询；停用后新规则不可再引用该预处理器。

### 3.8 智能作业场景编排规范
用于定义“提示命中后如何通过节点序列完成复杂作业”。

1. 编排对象结构（建议）
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `orchestration_id` | string | 是 | 编排唯一标识 |
| `orchestration_name` | string | 是 | 编排名称 |
| `status` | enum | 是 | `ENABLED` / `DISABLED` |
| `nodes` | array | 是 | 节点列表（按 `order` 执行） |
| `owner` | string | 是 | 责任人账号 |
| `updated_at` | datetime | 是 | 更新时间 |

2. 节点通用结构（建议）
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `node_id` | string | 是 | 节点唯一标识 |
| `node_type` | enum | 是 | `page_get` / `page_set` / `api_call` / `js_script` / `custom` |
| `order` | int | 是 | 节点执行顺序（从小到大） |
| `enabled` | boolean | 是 | 是否启用 |
| `input_mapping` | object | 否 | 输入参数映射（支持引用上下文） |
| `output_mapping` | object | 否 | 输出映射（写入上下文键） |
| `on_error` | enum | 否（平台内置） | 当前固定 `STOP`，前端不开放配置 |

3. 首批内置节点能力
- `page_get`：按 XPath 获取页面元素值并写入上下文。
- `page_set`：将上下文或固定值写入页面字段（XPath 定位）。
- `api_call`：引用已发布接口子项执行调用，参数支持固定值与页面/上下文取值映射。
- `js_script`：执行 JS 脚本逻辑，对输入进行计算/格式化，当前可按配置直接读写页面对象与上下文。

4. 执行与上下文规则
- 执行触发：仅由智能提示命中触发，独立触发入口不开放。
- 绑定关系：单个智能作业绑定一个编排定义，触发时执行该编排全链路。
- 执行顺序：按节点 `order` 串行执行，前序输出可被后序节点引用。
- 上下文作用域：单次编排实例内有效，实例结束后释放。
- 失败策略：当前固定为 `STOP`（节点失败即终止编排），不开放前端配置“继续执行”；失败事件写入运行日志，不影响业务主流程。
- 预览时机：若启用预览模式，则在编排节点执行完成后统一生成预览快照并弹窗确认，确认后一次性写入页面。
- 重复触发：悬浮按钮触发时重新启动一个新的编排实例。

5. 扩展与治理
- 支持 `custom` 节点类型扩展，需由系统管理员完成注册、发布与启停。
- 自定义节点需定义输入/输出 Schema、执行器实现与自测用例。
- 自定义节点发布前必须自测通过。
- 节点扩展停用后，新编排不可继续引用；历史执行日志可追溯。

## 4. 非功能需求

### 4.1 性能要求
1. 智能提示触发端到端 P95 <= 800ms。
2. 智能作业数据注入端到端 P95 <= 1500ms（不含内部系统数据源超时重试）。
3. 单页面连续操作场景下，JSSDK 不得造成明显卡顿（主线程长任务受控）。

### 4.2 安全要求
1. JSSDK 与服务端通信全链路 TLS。
2. 内部系统数据查询与注入须携带最小权限鉴权凭证。
3. 操作日志需满足审计留痕要求，不少于 180 天（可按行内规范调整）。
4. 个人与企业敏感信息按分级策略脱敏展示和存储。

### 4.3 兼容性要求
1. 首期支持银行内网标准浏览器环境（Chrome/Edge 主流版本）。
2. 适配主流业务系统页面技术栈（iframe/单页/多页）注入模式。
3. 网络波动时支持降级：提示能力可降级展示缓存规则，注入能力可降级为人工确认。

## 5. 边界与限制

### 5.1 In Scope
1. 智能提示三类能力：监管传达、风险提醒、新员工作业指引。
2. 智能作业：内部系统数据获取并注入当前页面。
3. JSSDK 注入运行、事件回传、基础效果统计。
4. 规则配置与灰度发布。

### 5.2 Out of Scope
1. 重构各业务系统页面与流程引擎。
2. 替代内部系统主数据平台或建设统一主数据平台。
3. 浏览器插件形态与桌面端独立客户端。

### 5.3 已知约束
1. 必须由业务系统提前注入 JSSDK 后才能生效。
2. 内部系统数据质量与可用性受上游系统影响。
3. 当前试点以开阳业务系统为主，入口按“专区 -> 业务菜单 -> 业务页面”组织。
4. 智能提示与智能作业由管理员在“业务菜单级”控制启停。
5. 各业务系统页面 DOM 结构差异较大，需要逐系统适配映射。

## 6. 验收标准（Given-When-Then）

1. 监管临时要求提示
- Given 规则运营已发布一条“开户业务监管临时要求”并配置触发页面
- When 审核人员进入目标页面并满足触发条件
- Then 页面在 800ms 内显示提示内容且记录命中日志

2. 智能提示条件关系运算
- Given 提示规则包含多个条件表达式，单条件为“左值+运算符+右值”，且左值/右值来源可选固定值、页面元素、接口值
- When JSSDK 获取页面元素值与接口值并执行条件运算
- Then OR 规则在任一条件满足时触发，AND 规则在全部条件满足时触发，并记录命中明细

3. 智能提示左右值预处理链
- Given 条件表达式配置了 `left_preprocessors` 与 `right_preprocessors`
- When 系统完成左右值取值并执行预处理
- Then 预处理器按各自链路顺序执行，处理结果参与条件运算并写入条件运算日志；首批支持“提取前几位”“数字转换”

4. DOM 变化触发提示
- Given 外部 SDK 已监听目标 DOM 节点变化并接入通知链路
- When DOM 值变化后外部 SDK 通知智能提示 SDK 执行规则运算
- Then 条件满足时立即弹出提示，条件不满足时不弹窗并记录判定日志

5. 提示联动作业触发
- Given 提示规则已配置“命中后执行智能作业”
- When 智能提示规则命中并完成提示渲染
- Then 系统自动执行对应智能作业并记录联动执行结果

6. 多提示合并展示
- Given 同一轮规则运算命中多条智能提示
- When 智能提示 SDK 进入渲染阶段
- Then 系统将多条提示合并到同一提示窗口展示

7. 联动作业顺序执行
- Given 同一轮命中多条且均配置联动作业的智能提示
- When 系统开始执行联动作业
- Then 联动作业按智能提示触发顺序依次执行并记录顺序日志

8. DOM 高频变化防抖
- Given 外部 SDK 在短时间内多次监听到同一目标 DOM 变化
- When 通知智能提示 SDK 执行规则
- Then 系统对触发进行防抖，避免重复弹窗与重复联动作业

9. 事中风险提醒
- Given 客户交易信息命中高风险规则
- When 用户点击提交前进入校验节点
- Then 系统展示高风险提醒，记录处理动作，并保持现有流程不变

10. 新员工作业指引
- Given 当前用户为新员工标记且在指定业务页面办理流程
- When 进入关键字段录入步骤
- Then 页面展示对应步骤指引与常见错误提示

11. 智能作业预览模式填充
- Given 智能作业规则已开启预览模式
- When 编排全部计算完成后系统展示待填充数据预览
- Then 用户可在预览中修改字段并确认，系统按确认后的值一次性写入页面

12. 提示控制下智能作业注入
- Given 提示规则已配置“命中后执行智能作业”且内部系统数据源可用
- When 用户操作导致智能提示规则命中并触发作业
- Then 系统将匹配字段注入当前页面并标记触发来源为“提示命中”

13. 预览模式取消填充
- Given 智能作业规则已开启预览模式
- When 用户在预览窗口点击取消
- Then 系统不执行注入并记录取消事件

14. 悬浮按钮重复触发
- Given 提示规则已配置悬浮按钮触发且该提示已命中
- When 用户点击页面悬浮按钮
- Then 系统触发该提示绑定作业并记录触发来源为“悬浮按钮”

15. 注入后不再处理
- Given 智能作业已完成字段注入
- When 页面后续出现同字段变化事件
- Then 系统不再对已注入字段执行后续处理

16. 注入异常可回退
- Given 内部系统数据查询超时或返回冲突值
- When 执行注入流程
- Then 系统不覆盖已填写关键字段，提示人工确认并记录失败原因

17. 接口子项预配置校验
- Given 提示条件表达式右值来源配置为接口返回值
- When 该规则引用未在接口子项中定义或未发布的接口
- Then 规则保存或发布失败，并返回“接口未预配置/未发布”的校验提示

18. 接口子项发布成功
- Given 管理员已完成接口子项定义并通过发布前校验
- When 将接口状态从 `DRAFT` 发布为 `PUBLISHED`
- Then 该接口可被提示条件表达式引用，并可追溯发布记录

19. 发布态接口引用生效
- Given 提示规则条件表达式引用了 `PUBLISHED` 状态接口
- When 规则触发并执行取值
- Then 系统成功获取接口值参与条件运算并记录接口标识

20. 页面参数依赖取值
- Given 接口参数映射中包含页面元素取值项
- When 提示命中后触发智能作业并准备调用接口
- Then 系统先按映射从页面提取参数并组装请求，提取成功后再发起接口调用

21. 页面参数缺失处理
- Given 接口请求依赖的必填页面参数为空或无法提取
- When 系统执行参数组装
- Then 本次接口调用跳过并记录“页面参数缺失”原因

22. 被引用接口下线拦截
- Given 某接口已被已发布规则引用
- When 用户尝试将该接口状态从 `PUBLISHED` 变更为 `OFFLINE`
- Then 系统拦截下线并返回“接口被引用，需先解除引用”

23. 接口发布下线权限校验
- Given 当前用户既不是接口 `owner` 也不是平台管理员
- When 尝试执行接口发布或下线操作
- Then 系统拒绝操作并返回无权限提示

24. 平台参数统一控制校验
- Given 用户在接口定义时尝试提交自定义 `timeout_ms` 或 `retry_count`
- When 系统处理接口保存/发布请求
- Then 系统忽略用户提交值并应用平台统一参数策略

25. 前端参数项隐藏校验
- Given 用户进入接口定义与编辑页面
- When 查看接口配置表单
- Then 页面不展示 `timeout_ms`、`retry_count` 可编辑项

26. 参数映射测试值生效
- Given 参数映射项已配置 `test_value` 且当前为测试模式
- When 系统执行接口参数组装
- Then 该参数优先使用 `test_value` 参与请求组装

27. 参数类型默认字符串
- Given 参数映射项未配置 `value_type`
- When 系统完成参数取值
- Then 系统按字符串类型处理该参数

28. 日志落库与查询脱敏
- Given 参数取值与接口调用已执行
- When 查看运行日志详情
- Then 系统保存全量日志记录，查询展示按敏感规则脱敏

29. 首批预处理器-提取前几位
- Given 左值预处理器链配置 `prefix_extract(length=6)` 且原始值为 `"6222021234567890"`
- When 系统执行条件预处理
- Then 左值输出 `"622202"` 并参与条件运算

30. 首批预处理器-数字转换
- Given 右值预处理器链配置 `to_number` 且原始值为 `"00123.40"`
- When 系统执行条件预处理
- Then 右值输出 `123.4` 并参与条件运算

31. JS 预处理器异常隔离
- Given 某 JS 预处理器执行抛出异常或超时
- When 系统执行该条件预处理
- Then 当前条件按不满足处理并记录异常，不影响同规则其他条件继续计算

32. 管理员扩展预处理器发布
- Given 当前用户为系统管理员并提交新的 JS 预处理器定义
- When 通过测试值验证与安全检查后发布
- Then 该预处理器可被新规则引用，且可被管理员后续启停

33. 智能作业场景编排顺序执行
- Given 某提示规则命中后绑定一个包含 4 个节点的智能作业编排（页面取值、接口调用、JS 脚本、页面赋值）
- When 系统触发该编排实例
- Then 节点按配置顺序串行执行，且每个节点执行结果可被后续节点引用

34. 页面取值与赋值节点执行
- Given 编排中配置 `page_get` 与 `page_set` 节点，且 XPath 可解析
- When 编排执行到对应节点
- Then 系统成功读取页面值并在后续节点将值写入目标字段，同时记录节点输入输出日志

35. 编排接口调用节点执行
- Given 编排 `api_call` 节点引用已发布接口子项，参数映射包含页面取值项
- When 执行到接口调用节点
- Then 系统按映射完成参数组装并发起调用，返回值写入上下文供后续节点使用

36. 编排脚本节点执行（当前不限制页面写入）
- Given 编排包含 `js_script` 节点且脚本需要对接口结果做格式化
- When 执行脚本节点
- Then 脚本执行后可输出格式化结果并可按配置直接更新页面对象，且执行行为被记录

37. 编排失败终止与主流程隔离
- Given 编排中某节点因缺失必需输入执行失败
- When 系统处理该失败
- Then 当前编排实例终止并记录失败节点与原因，业务主流程继续执行不被阻断

38. 自定义节点扩展生效
- Given 系统管理员发布了新的 `custom` 节点类型并处于启用状态
- When 规则运营在编排中引用该节点并由提示触发执行
- Then 系统可识别并执行该节点；若节点被停用，则新编排保存或发布时被拦截

39. 编排失败策略固定为 STOP
- Given 编排中某节点执行失败
- When 系统处理节点失败
- Then 系统终止当前编排实例且不提供“继续执行”分支，失败原因全量留痕

40. 悬浮按钮触发新实例
- Given 用户已通过悬浮按钮执行过一次编排并产生实例上下文
- When 用户再次点击同一悬浮按钮
- Then 系统创建新的编排实例并重新执行，不复用上一次实例上下文

41. 单作业绑定单编排
- Given 某智能作业已绑定一个 `orchestration_id`
- When 提示命中触发该智能作业
- Then 系统执行该 `orchestration_id` 对应的编排全链路，不并发执行其他编排

42. 编排预览统一时机
- Given 智能作业已绑定编排且开启预览模式
- When 编排所有节点执行完成并产出待写入字段
- Then 系统统一弹出一次预览窗口，用户确认后一次性写入页面
