# 智能作业产品文档（Job PRD）

> 文档状态：生效
> 更新时间：2026-03-17
> 归属：营小助配置中心（智能作业子文档）
> 关联总纲：`spec/config-center-product/prd-analysis.md`

## 1. 文档目标

沉淀智能作业子系统的独立产品口径，覆盖场景管理、编排模型、节点能力、执行方式和控制约束。

## 2. 范围

### 2.1 In Scope

1. 作业场景基础信息维护（名称、执行方式、状态、责任归属）。
2. 场景编排主界面（左-中-右布局：节点库、画布、属性面板）。
3. 节点添加、连线、排序、属性配置、保存。
4. 执行方式配置（无提示自动执行、提示后自动执行、预览确认后执行、悬浮按钮触发）。
5. 预览确认页产品口径。
6. 悬浮按钮与二次触发规则。
7. 失败兜底、审计留痕与发布控制口径。

### 2.2 Out of Scope

1. 执行实例列表与执行明细详情页。
2. 完整联调测试能力与外部系统仿真能力。
3. 多人实时协同编排。

## 3. 核心产品口径

1. 智能作业是平台核心能力，不是外围插件能力。
2. 规则与作业场景分开建模、分开发布控制，避免对象耦合。
3. 编排能力面向业务可理解，避免无约束低代码化。
4. 执行失败默认可感知、可回退、可追溯。
5. 自动化动作必须绑定责任归属、适用范围、有效期与审计留痕。

## 4. 场景模型与对象关系

### 4.1 核心场景框架

1. 页面字段查数并预填当前页面。
2. 审核过程中的辅助取数与比对。
3. 高频重复录入场景的半自动填写。
4. 自动触发后允许悬浮按钮二次触发复用。
5. 失败后允许重试且不复用脏上下文。

### 4.2 规则与作业对象关系

1. 规则负责定义“什么时候触发”。
2. 作业场景负责定义“触发后做什么”。
3. 两者分别版本化、分别发布控制，不合并为同一对象。
4. 规则可关联作业场景，场景也可被多个规则引用，关系必须可追溯。

### 4.3 场景标识口径

1. 当前仅保留场景名称。
2. 场景名称执行常规校验（非空、长度、平台内唯一、非法字符约束）。
3. 当前不单独建设场景编码字段。

## 5. 编排模型与节点能力

### 5.1 编排页面结构

作业模块页面结构采用五段式：

1. 场景基础信息区。
2. 触发关联信息区。
3. 节点编排区。
4. 执行方式与预览配置区。
5. 校验与发布区。

### 5.2 编排交互模型

1. 节点编排区采用“流程图主视图 + 结构化表单属性编辑”组合。
2. 不采用纯表格式编排，也不采用完全自由、无约束画布交互。
3. 节点选中后在右侧属性面板编辑配置。
4. 支持一键自动排版提升大场景可读性。

### 5.3 基础节点模型

1. `page_get`：页面取值。
2. `api_call`：接口调用。
3. `list_lookup`：名单检索。
4. `js_script`：加工处理。
5. `page_set`：页面写值。

以上构成最小可用动作链：取值 -> 查名单/查数 -> 加工 -> 回填。

`list_lookup` 节点产品口径：

1. 节点引用平台统一的名单数据资产，不单独维护 Excel 文件。
2. 节点配置复用名单选择、匹配字段、输入来源三类公共配置。
3. 节点最小输出结果为：是否命中、名单名称，以及写入 `resultKey` 的结构化结果。
4. 若名单未发布、已过期或匹配字段失效，节点保存与发布均需阻断。

### 5.4 节点保存与排序规则

1. 连线拓扑关系决定主执行顺序。
2. 拓扑无法完全排序时，按横向位置兜底排序。
3. 节点布局位置写入配置，支持重开还原。
4. 节点需可见输入、输出、失败状态与耗时。

### 5.5 测试能力边界

1. 平台仅承诺基础校验能力。
2. 真实链路验证、外部接口联调、页面注入结果确认，仍在外部系统或真实业务环境完成。

## 6. 执行方式与预览确认

### 6.1 执行方式口径

1. 无提示自动执行。
2. 提示后自动执行。
3. 预览确认后执行。
4. 悬浮按钮手动触发。

前三类属于执行方式，悬浮按钮属于补充触发入口。
5. 默认行为为“自动执行 + 直接执行”，列表中不单独展示；仅展示特殊行为（预览确认、按钮触发、补充按钮）。

### 6.2 预览确认页口径

1. 主视图采用字段对照表。
2. 展示字段名称、原值、拟写入值、数据来源。
3. 支持按字段勾选是否写入。
4. 异常字段单独标红并说明原因。
5. 顶部展示作业摘要、执行结果概览与必要风险提示。

## 7. 悬浮按钮与二次触发

1. 悬浮按钮是规则联动之外的第二触发入口。
2. 二次触发必须新建执行实例，不复用旧实例上下文。
3. 仅在当前页面存在可执行场景时展示。
4. 默认不在所有页面常驻展示。
5. 同一页面默认保留一个主悬浮入口。
6. 即使自动执行完成，仍允许用户手动再次触发。
7. 悬浮按钮支持配置按钮文案与坐标位置（X/Y 百分比）。
8. 按钮触发场景下悬浮按钮配置为必填；自动执行场景下可选启用补充按钮。

## 8. 发布控制与运行边界

1. 自动化强度可由业务侧配置，责任归属通过配置记录与发布审计共同承接。
2. 自动化场景发布前必须具备责任人、范围、有效期与留痕，不再要求独立风险确认动作。
3. 失败默认停止并提示，不静默吞错。
4. 失败后必须可回到手工路径。
5. 执行过程默认实例化，保证审计追溯。

## 9. 验收标准

1. 可完成场景创建、节点编排、属性配置、保存发布全流程。
2. 四类基础节点可在画布中正确编排并形成执行顺序。
3. 执行方式四类口径可配置并按规则生效。
4. 悬浮按钮触发符合可见性与新实例规则。
5. 不出现执行实例列表与独立联调测试面板。
6. 关键配置与状态变更可在审计中检索。

## 10. 依赖关系

1. 页面资源中心：字段目标映射基础。
2. API注册中心：接口调用节点依赖。
3. 预处理器中心：数据加工能力依赖。
4. 发布与审计中心：发布、状态与追溯能力依赖。
