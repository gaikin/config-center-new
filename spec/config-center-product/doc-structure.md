# 配置中心产品文档拆分方案

> 更新时间：2026-03-12
> 目的：明确“平台总纲 + 智能提示 + 智能作业”三层文档结构，降低后续需求冲突。

## 1. 结论
建议拆分为三份产品文档：
1. 平台总纲（统一术语、共享模型、角色权限、发布控制、审计口径）
2. 智能提示产品文档（规则、触发、提示交互、关闭策略）
3. 智能作业产品文档（场景编排、节点模型、执行方式、预览确认）

## 2. 为什么要拆
1. 提示和作业迭代节奏不同，合并文档容易互相覆盖。
2. 评审对象不同，拆分后责任边界更清晰。
3. 前端与后端实现口径更容易对齐，减少“文档描述正确但落地歧义”。

## 3. 拆分后如何避免割裂
1. 跨模块规则只在平台总纲维护，子文档引用不重复定义。
2. 子文档变更需同步更新总纲的影响边界与变更记录。
3. 发布校验、权限、审计等平台级规则统一由总纲主责。
