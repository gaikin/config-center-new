# 智能提示产品文档（Sentinel PRD）

> 文档状态：生效
> 更新时间：2026-03-12
> 归属：营小助配置中心（智能提示子文档）
> 关联总纲：`spec/config-center-product/prd-analysis.md`

## 1. 文档目标

沉淀智能提示（哨兵）子系统的独立产品口径，覆盖规则配置、条件判断、提示展示、触发日志与控制约束。

## 2. 范围

### 2.1 In Scope

1. 规则基础信息维护（名称、页面资源、优先级、状态、适用范围）。
2. 条件编辑（单层条件、全局 AND/OR、左右值配置）。
3. 条件来源管理（页面字段、API字段、名单字段、上下文变量、固定值）。
4. 提示展示与关闭策略（静默、浮窗、关闭方式）。
5. 提示与作业入口关系（单入口联动边界）。
6. 提示文案配置体验。
7. 规则启停、版本控制、审计追踪。
8. 条件值来源样式规范与设计稿标注清单。

### 2.2 Out of Scope

1. 多层条件组嵌套编辑。
2. 条件独立“手工预览”调试面板。
3. 作业节点编排细节与执行实例管理。

## 3. 核心产品口径

1. 条件模型采用单层结构，不开放父子组嵌套。
2. 全部条件共享一个全局逻辑关系：AND 或 OR。
3. 条件组编辑区采用左右布局：左侧为“左值 / 逻辑运算 / 右值”，右侧为单一属性面板。
4. 右侧属性面板不设置固定页签；选中左值或右值时切换显示对应属性。
5. 属性面板字段统一包括：值类型、值、入参配置（接口）、预处理器。
6. 预处理器采用“点击添加条目 + 参数输入”方式配置。
7. 当值类型不需要接口入参时，隐藏入参配置区。
8. 左值与右值来源统一收敛为：页面字段、API字段、名单字段、上下文变量、固定值。
9. API 来源展示格式统一为“API名称.输出值路径”。
10. 多提示命中采用串行执行策略，不并发堆叠。
11. 提示仅支持静默提示和浮窗提示两种类型。

## 4. 条件编辑与规则模型

### 4.1 条件组与属性面板交互

1. 左侧条件编辑保持固定顺序，保证业务人员阅读一致性。
2. 右侧属性面板只显示当前选中对象配置，避免左右配置割裂。
3. 切换左值/右值对象时，右侧保留对象标题与当前来源类型提示。
4. 来源切换后若存在旧配置残留，必须弹确认并执行清理。
5. API名称或输出值路径缺失时，显示“API名称.(未绑定输出值)”并阻断保存。

### 4.2 条件值来源样式规范

#### 4.2.1 来源类型视觉字典

| 来源类型 | 标签文案 | 图标建议 | 颜色语义（建议 token） | 展示示例 |
|------|------|------|------|------|
| 固定值 | 固定值 | 数值/文本图标 | 中性灰（`--cc-source-fixed`） | `固定值 1000` |
| 页面元素 | 页面元素 | 页面/字段图标 | 信息蓝（`--cc-source-page`） | `页面元素 客户等级` |
| API | API | 接口/网络图标 | 能力绿（`--cc-source-api`） | `API CustomerInfo.data.level` |
| 名单字段 | 名单字段 | 名单/检索图标 | 紫色（`--cc-source-list`） | `名单字段 高风险客户名单.risk_level` |
| 上下文变量 | 上下文变量 | 变量/上下文图标 | 提示橙（`--cc-source-context`） | `上下文变量 当前用户角色` |

#### 4.2.2 展示规则

1. 编辑态与摘要态均采用“标签 + 值内容”双信息结构，不允许仅显示技术路径。
2. 编辑态标签固定在值内容前，值内容过长单行省略，悬浮展示完整文本。
3. 摘要态优先展示可读名称，技术路径作为次级信息展示。
4. API来源统一展示“API名称.输出值路径”，不展示内部 ID。
5. API 缺少输出值路径时进入告警态并阻断保存。

#### 4.2.3 交互状态样式规则

| 状态 | 样式要求 |
|------|----------|
| 默认态 | 展示来源标签与主值内容；标签使用来源语义色。 |
| 悬浮态 | 标签与边框轻度强化，不改变来源语义。 |
| 选中态 | 当前对象高亮，右侧属性面板同步定位。 |
| 告警态 | 标签切换告警样式并显示原因文案。 |
| 禁用态 | 保留来源识别能力，但降低强调度。 |

#### 4.2.4 可访问性与一致性要求

1. 来源识别必须同时依赖图标与文案，不允许仅依赖颜色。
2. 标签文字与背景需满足可读性要求。
3. 左值与右值、编辑态与摘要态使用一致图标和语义色。
4. 窄屏下可换行，但“标签 + 值内容”关系不可拆散。

#### 4.2.5 验收标准

1. 任意条件在 2 秒内可识别左值和右值来源类型。
2. API来源在列表与编辑态均展示“API名称.输出值路径”且与配置一致。
3. 缺失 API 输出值路径时，保存被阻断并出现可理解提示。
4. 来源切换后无旧来源脏配置残留。

### 4.3 条件值来源样式标注清单（设计稿）

#### 4.3.1 文本与尺寸标注

| 标注项 | 建议值 | 说明 |
|------|------|------|
| 来源标签字号 | 12px | 用于来源标签文案。 |
| 来源标签字重 | 500 | 保证标签识别度。 |
| 值内容字号 | 13px | 用于主值展示。 |
| 值内容字重 | 400 | 与标签形成层级。 |
| 行高 | 20px | 保证可读性与密度平衡。 |
| 条件行最小高度 | 32px | 保证垂直居中。 |

#### 4.3.2 间距与圆角标注

| 标注项 | 建议值 | 说明 |
|------|------|------|
| 标签左右内边距 | 8px / 8px | 标签左右留白一致。 |
| 标签高度 | 22px | 与字号匹配。 |
| 标签圆角 | 11px | 胶囊标签形态。 |
| 图标与文案间距 | 4px | 图文紧凑可辨识。 |
| 标签与值内容间距 | 8px | 保持来源与内容分区。 |
| 容器左右内边距 | 10px / 10px | 保持条件行对齐。 |
| 容器圆角 | 6px | 兼顾密度与可点击感。 |

#### 4.3.3 图标标注

| 标注项 | 建议值 | 说明 |
|------|------|------|
| 来源图标尺寸 | 14px x 14px | 四类来源统一尺寸。 |
| 图标描边风格 | 1.5px 线性 | 与平台通用图标风格一致。 |
| 图标垂直对齐 | 与文案基线居中 | 避免视觉抖动。 |

#### 4.3.4 状态色值标注（建议）

1. 若设计系统已有语义色 token，优先复用系统 token。
2. 临时色值建议如下：

| 来源类型/状态 | token | 建议前景色 | 建议背景色 | 建议边框色 |
|------|------|------|------|------|
| 固定值 | `--cc-source-fixed` | `#475467` | `#F2F4F7` | `#D0D5DD` |
| 页面元素 | `--cc-source-page` | `#175CD3` | `#EFF8FF` | `#B2DDFF` |
| API | `--cc-source-api` | `#027A48` | `#ECFDF3` | `#ABEFC6` |
| 名单字段 | `--cc-source-list` | `#7A00E6` | `#F4EBFF` | `#D9B8FF` |
| 上下文变量 | `--cc-source-context` | `#B54708` | `#FFFAEB` | `#FEC84B` |
| 告警态 | `--cc-source-warning` | `#B42318` | `#FEF3F2` | `#FDA29B` |
| 选中态 | `--cc-source-selected` | `#175CD3` | `#EFF8FF` | `#84CAFF` |

#### 4.3.5 交互与动效标注

1. 悬浮态：120ms 过渡，边框和背景轻度增强。
2. 选中态：120ms 过渡，外框高亮并同步右侧聚焦。
3. 告警态：不使用闪烁，通过颜色、图标、文案三重提示。
4. 来源切换触发清理确认时，保持当前行选中态。

#### 4.3.6 切图与实现交付要求

1. 提供四类来源在默认/悬浮/选中/告警/禁用五态样例。
2. 提供左值与右值各一套样例，证明双侧一致。
3. 在设计工具中标注 token 名称，不仅标注色值。
4. 前端按 token 接入，不在组件内写死十六进制颜色。
5. 交付包包含组件标注页、状态矩阵页、异常样例页。

### 4.4 条件校验边界

1. 配置阶段提供必填校验、值类型校验、引用有效性校验。
2. 不自动带入历史命中数据或线上真实样本作为测试输入。
3. 配置页校验目标是保障可配置性与可发布性，不替代真实运行验证。
4. 真实链路验证与跨系统联调仍应在外部系统或实际业务环境完成。

### 4.5 规则模型口径

规则对象至少覆盖以下维度：

1. 规则名称。
2. 规则绑定页面。
3. 条件集合与逻辑关系。
4. 左值和右值来源。
5. 预处理链。
6. 优先级。
7. 生效范围与有效期。
8. 状态。
9. 提示配置。
10. 是否联动作业。

规则标识当前仅保留规则名称，名称执行常规校验并保持平台内全局唯一。

### 4.5.1 页面字段引用口径

1. 条件来源统一收敛为“页面字段、API字段、名单字段、上下文变量、固定值”五类。
2. 当来源为页面字段时，需先选择“公共字段”或“页面特有字段”，再引用页面资源中已建档字段。
3. 公共字段的引入目标首先是保证条件可迁移；若需复用整条规则，应额外提供“规则模板 -> 页面规则”的复用链路，而不是把“公共字段”本身当成规则复用机制。
4. 规则模板不绑定具体页面，且仅允许引用公共字段；模板复用到目标页面后，生成独立页面规则，再按需补充页面特有字段。
5. 业务人员的主流程应始终是“新建页面规则”；模板仅作为新建时的快捷来源，不要求用户先专门进入模板维护流程。
6. 建议交互顺序为：选择页面资源 -> 可选套用模板 -> 自动带入条件与提示配置 -> 保存页面规则 -> 再按需微调。
7. 可提供独立“规则模板中心”供专家角色沉淀高复用模板，但该中心属于增强入口，不替代业务人员的日常规则配置入口。
8. 页面字段在页面资源层必须携带元素类型，至少区分 `INPUT`、`SELECT`、`TEXTAREA`、`BUTTON`、`READONLY_TEXT`。
9. `READONLY_TEXT` 类型默认以 `textContent` 作为取值来源，用于承接 `span/p` 等只读文本节点。
10. 页面字段取值结果至少区分 `ABSENT`、`EMPTY`、`VALUE` 三态；其中 `ABSENT` 表示元素不存在，条件结果记为 `INVALID`，规则层默认视为不命中。

## 5. 提示展示与关闭策略

1. 当前阶段仅支持静默提示和浮窗提示两种展示方式。
2. 提示关闭行为支持计时自动关闭、默认手工关闭、计时后手工关闭。
3. 多提示命中按串行策略处理，展示顺序按规则优先级。
4. 同一规则在同一页面稳定状态下默认不重复打扰。
5. 提示关闭后，当前规则在本次页面停留中不再继续执行后续行为。

## 6. 提示语义、入口与文案配置

### 6.1 提示语义口径

1. 提示统一作为规则命中后的信息承载方式。
2. 不按“监管类提示/风险类提示”等类型做独立产品建模。
3. 业务语义通过命中上下文与具体文案表达。

### 6.2 提示与作业入口关系

1. 提示可承载单一作业引导入口，但不承担多作业分发。
2. 是否进入作业、何时进入作业以及是否需要预览确认，由作业场景的执行方式配置决定。
3. 当执行方式需要提示承接时，提示入口统一为“确认”按钮；用户点击确认后进入对应作业链路。
4. `无提示自动执行` 不依赖提示入口，`悬浮按钮手动触发` 由悬浮入口单独承接。
5. 多作业或复杂场景由悬浮按钮或独立作业入口承接。

### 6.3 文案配置体验

1. 提示文案由业务人员按场景自定义编写。
2. 当前阶段不强制模板化复用与结构化文案模型。
3. 配置重点在展示方式、展示时机与触发条件。

## 7. 生命周期与发布控制

1. 规则状态（待发布、生效中、已停用、已失效）可追溯。
2. 规则状态变更必须记录操作人、时间、对象、结果。
3. 引用资源失效（API注册/预处理器等）时禁止发布。

## 8. 验收标准

1. 可在单层条件模型中完成新增、编辑、删除。
2. 全局 AND/OR 可影响全部条件组合结果。
3. 左右值编辑与右侧单属性面板联动正确。
4. 来源样式、状态样式与设计稿标注清单一致。
5. 提示展示、关闭、多提示串行策略按口径生效。
6. 规则启停与配置变更可在审计中检索。

## 9. 依赖关系

1. 页面资源中心：提供页面与字段元数据。
2. API注册中心：提供 API 字段来源。
3. 预处理器中心：提供复用预处理能力。
4. 发布与审计中心：提供发布与追溯能力。
