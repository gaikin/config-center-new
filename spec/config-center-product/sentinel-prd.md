# 智能提示产品文档（Sentinel PRD）

> 文档状态：生效
> 更新时间：2026-03-12
> 归属：营小助配置中心（智能提示子文档）
> 关联总纲：`spec/config-center-product/spec.md`

## 1. 文档目标

沉淀智能提示（哨兵）子系统的独立产品口径，覆盖规则配置、条件判断、提示展示、触发日志和治理约束。

## 2. 范围

### 2.1 In Scope

1. 规则基础信息维护（名称、页面资源、优先级、状态、适用范围）。
2. 条件编辑（单层条件、全局 AND/OR）。
3. 条件左右值来源（页面字段、接口字段、上下文字段、常量）。
4. 预处理器链（左右值分别支持下拉多选）。
5. 提示展示与关闭策略（静默、浮窗、关闭方式）。
6. 规则启停、版本治理和审计追踪。

### 2.2 Out of Scope

1. 多层条件组嵌套编辑。
2. 条件“手工预览”独立调试面板。
3. 作业节点编排细节与执行实例管理。

## 3. 核心产品口径

1. 条件模型采用单层结构，不开放父子组嵌套。
2. 全部条件共享一个全局逻辑关系：AND 或 OR。
3. 单条条件编辑顺序固定为：
   左值类型 -> 选择值 -> 左预处理器 -> 逻辑运算 -> 右值类型 -> 选择值 -> 右预处理器。
4. 预处理器交互为条件行内下拉多选，不使用弹窗模式。
5. 当运算符为 EXISTS 时，右值与右预处理器禁用。

## 4. 基础信息配置规则

1. 规则名称必填且全局唯一。
2. 页面资源必选。
3. 优先级范围 1-999。
4. `TIMER_THEN_MANUAL` 时必须填写超时秒数，且大于 0。
5. 关闭方式与确认按钮配置遵循统一治理约束。

## 5. 交互说明

### 5.1 规则中心

1. 提供规则列表、状态切换、基础编辑入口、条件编辑入口。
2. 支持按规则状态查看与治理操作。

### 5.2 条件编辑抽屉

1. 顶部配置整体逻辑（AND / OR）。
2. 条件表格支持新增、删除、行内编辑。
3. “选择值”列优先展示并加宽，突出核心输入。
4. 左右预处理器均支持行内多选。

## 6. 生命周期与治理

1. 草稿、发布、停用、过期状态可追溯。
2. 规则状态变更必须记录操作人、时间、对象和结果。
3. 引用资源失效（接口/预处理器等）时禁止发布。

## 7. 验收标准

1. 可在单层条件表格中完成条件新增、编辑、删除。
2. 全局 AND/OR 可影响所有条件组合结果。
3. 预处理器可通过下拉多选配置并正确保存。
4. `TIMER_THEN_MANUAL` 未填超时时间时无法保存。
5. 规则启停与配置变更可在审计中检索。

## 8. 依赖关系

1. 页面资源中心：提供页面和字段元数据。
2. 接口定义中心：提供接口字段来源。
3. 预处理器中心：提供可复用预处理能力。
4. 治理与审计中心：提供发布与追溯能力。
