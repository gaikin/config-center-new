### D. 跨模块实现防漏检查清单（研发执行）

适用触发条件：
- 新增或修改“取值来源/字段映射/条件引用/API出入参”任一能力。

必做检查（提交前全部完成）：
1. 对象模型：类型定义与存储结构已补齐，避免只改 UI 不改模型。
2. 配置 UI：所有消费该能力的页面都可配置（例如 API注册、条件编辑、作业节点）。
3. 保存结构：前端提交 payload 与 mock/service 解析字段一致，可回显可编辑。
4. 运行解析：运行/预览逻辑可识别新字段（含兜底和异常提示）。
5. 验收用例：至少覆盖“可配置、可保存、可回显、可执行、异常阻断”五类场景。

强制门禁（PR 自检项）：
1. `rg` 全局检索关键字段，确认所有消费点已联动更新。
2. 本地 `npm run build` 必须通过。
3. 在 PR 描述附“联动影响面清单”（模块、文件、校验结果）。
