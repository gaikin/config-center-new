 前端交互问题清单                                                                                                    …

  1. API注册页面                                                                                                        
  - 问题1.1：表单字段ID可以手动编辑，但应该由系统自动生成（第840-842行）                                                  - 问题1.2：Body JSON模板和输入参数表单是分离的，用户容易混淆两者关系
  - 问题1.3：出参配置中"路径"字段需要手动输入，没有提供路径选择器或自动补全功能
  - 问题1.4：调试功能在Modal中，但编辑在Drawer中，交互不一致弹窗混杂
  - 问题1.5：测试和生产路径切换需要手动选择，没有环境切换的明显引导

  2. 角色管理页面

  - 问题2.1：成员分配使用纯文本域，没有提供成员选择的下拉或搜索功能
  - 问题2.2：操作类型（actions）是多选Select，但没有说明每个权限的含义
  - 问题2.3：成员数是手动输入的，应该根据实际成员数自动计算（第258-260行）
  - 问题2.4：角色类型没有对应的权限预设，新增角色时不知道该选哪个类型

  3. 预处理器中心页面

  - 问题3.1：ID字段可以手动编辑（第149-151行），应该系统自动生成
  - 问题3.2：被引用次数可以手动编辑，应该由系统自动统计
  - 问题3.3：预处理器类型切换（内置/脚本）时，没有对应的配置面板切换
  - 问题3.4：缺少脚本编辑器，脚本类型预处理器应该提供代码编辑能力

  4. 页面资源中心页面

  - 问题4.1：ID字段可编辑（第291-293行），应自动生成
  - 问题4.2：元素数量可编辑（第321-323行），应自动计算
  - 问题4.3：站点和菜单筛选是级联的，但切换站点时菜单筛选没有自动重置（需用户手动）
  - 问题4.4：识别口径是纯文本输入，没有提供规则构建器或选择器
  - 问题4.5：元素映射Drawer中删除元素有确认，但编辑没有，操作确认不一致

  5. 智能作业页面

  - 问题5.1：风险确认状态可以手动编辑（第718行），应该是只读的
  - 问题5.2：作业编排Drawer中有多个Card区域，但没有清晰的视觉层级区分
  - 问题5.3：节点属性编辑在右侧面板，保存按钮位置不明显（第888行）
  - 问题5.4：ReactFlow画布和节点库在同一Card内，拖拽体验可能不佳
  - 问题5.5：预览确认中异常字段有视觉标记，但没有悬停提示解释异常原因
  - 问题5.6：悬浮按钮触发功能缺少确认，可能误触发

  6. 通用问题

  - 问题6.1：所有页面的"更新时间"字段都是只读显示，但编辑时没有明确标注
  - 问题6.2：状态切换（启用/停用）都是直接的Button，没有二次确认
  - 问题6.3：分页pageSize固定为6，没有提供分页大小选择
  - 问题6.4：表单Modal/Drawer关闭时没有检测未保存更改的提示
  - 问题6.5：表格操作列按钮过多时没有折叠处理
  - 问题6.6：缺少加载状态在表格之外的地方提示
