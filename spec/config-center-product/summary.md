# 营小助配置中心文档摘要

## 1. 产品定位
营小助配置中心是银行内部页面级智能辅助平台，以 JSSDK 方式嵌入业务系统页面，为运营与审核人员提供智能提示与智能作业能力。

## 2. 核心能力
1. 智能提示
- 统一提示引擎，业务分类包括监管临时要求传达、事中风险提醒、新员工作业指引。
- 触发方式为多来源取值规则运算。
- 每条提示由一组条件构成，条件关系支持“任意满足（OR）/全部满足（AND）”。
- 条件表达式采用“左值 + 运算符 + 右值”，左值和右值均支持固定值、页面元素、接口返回值。
- 条件表达式左值与右值均支持独立预处理器链，按顺序执行后参与运算（首批：提取前几位、数字转换）。
- 预处理器后续由系统管理员可随时扩展（新增/启停）。
- 预处理器统一采用 JavaScript 实现并在平台沙箱中执行。
- 接口来源值必须先在“接口配置子项”完成定义并发布后，规则才可引用。
- 接口子项采用字段化模型与状态流转（`DRAFT`/`PUBLISHED`/`OFFLINE`），仅发布态可被引用。
- 接口版本策略当前不启用但保留扩展字段；被引用接口不允许直接下线；接口发布/下线仅 owner 或管理员可操作。
- `timeout_ms` 与 `retry_count` 由平台统一控制，不对用户开放配置，前端页面不展示配置项。
- 部分接口参数可配置为页面取值来源，运行时按映射先取页面值再组装接口请求。
- 参数映射项新增 `test_value` 用于联调模拟取值。
- 参数类型默认按字符串处理（未配置 `value_type` 时）。
- 接口参数映射层不使用预处理器链，保持参数组装逻辑简化。
- 参数与调用日志全量落库，查询与展示按敏感规则脱敏。
- DOM 定位方式统一为 XPath，条件运算符支持常见运算符集合。
- 由外部 SDK 监听 DOM 变化并通知智能提示 SDK 执行规则，条件满足即弹窗。
- 同轮多提示命中时，合并到同一提示窗口展示。
- 生效策略采用 IP + 人员 + 机构混合生效。

2. 智能作业
- 从内部系统数据源获取数据。
- 按字段映射规则注入当前页面。
- 触发完全由智能提示控制，不提供独立触发入口。
- 联动作业在提示渲染后触发，多个联动作业按提示触发顺序执行。
- 支持预览模式：编排全部计算完成后统一展示待填充数据，用户可修改并确认后一次性写入。
- 预览模式下取消填充则不注入，并记录取消事件。
- 支持提示控制下的页面悬浮按钮触发，便于用户重复执行该提示绑定的智能作业。
- 支持场景编排：单个智能作业可由一组节点串联执行，默认按顺序运行并传递上下文。
- 绑定关系为“一作业一编排”：每个智能作业绑定一个编排定义执行全链路。
- 首批编排节点包含：获取页面元素值、页面赋值、接口调用、执行 JavaScript 脚本。
- 编排失败策略当前固定为 `STOP`（节点失败即终止该编排实例），不开放前端“继续执行”配置。
- 悬浮按钮每次点击均启动全新编排实例，不复用上次实例上下文。
- `js_script` 节点当前可按配置直接读写页面对象与上下文。
- 编排节点支持扩展，自定义节点由系统管理员发布与启停。
- 自定义节点需自测通过后方可发布。
- 注入完成后不再进行后续处理。
- 冲突或异常时不改变现有流程，保留人工处理与记录。

## 3. 运行与治理
1. 运行环境为浏览器，由业务系统提前注入 JSSDK。
2. 启用粒度为业务菜单级，由管理员控制智能提示和智能作业启停。
3. 安全合规最小集：敏感信息脱敏、审计日志留痕。
4. 仅采集智能提示与智能作业运行数据。

## 4. 范围边界
1. 不重构业务系统核心流程与页面主架构。
2. 不替代内部主数据平台。
3. 风险提醒仅做提示与记录，不改变现有流程。
