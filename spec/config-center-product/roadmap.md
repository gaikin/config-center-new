# 营小助配置中心 V1 功能优先级与里程碑计划
> 版本：V1.0（P0 已确认）  
> 说明：按阶段推进，不绑定具体时间与迭代排期

## 1. 优先级分层

## 1.1 P0（V1 必须上线）
1. JSSDK 基础运行框架
- 页面预注入初始化
- 上下文采集（系统/页面/角色/关键字段）
- 服务端请求与结果回传链路

2. 智能提示核心能力
- 统一提示引擎（业务分类包含：监管临时要求传达、事中风险提醒、新员工作业指引）
- 提示触发、优先级和展示策略可配置
- 支持基于 DOM 取值的条件运算，条件关系支持“任意满足（OR）/全部满足（AND）”
- 条件表达式支持“左值+运算符+右值”，且左值/右值来源支持固定值、页面元素、接口返回值
- 接口来源值必须先在配置管理子项完成定义与发布，再允许规则引用
- 接口子项采用字段化配置模型与状态机（`DRAFT`/`PUBLISHED`/`OFFLINE`）
- DOM 定位方式统一为 XPath，条件运算符支持常见运算符集合
- 由外部 SDK 监听 DOM 变化并通知智能提示 SDK 执行规则，条件满足触发弹窗

3. 智能作业核心能力
- 内部系统数据查询统一封装
- 字段映射规则执行
- 页面自动注入与状态反馈
- 智能作业触发完全由智能提示控制（不提供独立触发入口）
- 支持预览模式（编排完成后统一预览、可修改、确认后一次性写入）
- 支持提示控制下页面悬浮按钮触发（用于重复执行已绑定作业）
- 支持场景编排（由节点序列定义智能作业执行流）
- 一作业一编排（每个智能作业绑定单一编排定义）
- 首批内置节点：页面取值、页面赋值、接口调用、JavaScript 脚本执行
- 编排失败策略固定为 `STOP`（失败即终止，不开放继续执行配置）
- 悬浮按钮重复触发时每次启动新编排实例（不复用上下文）
- `js_script` 节点当前可按配置直接读写页面对象与上下文
- 节点类型支持扩展（管理员发布与启停）
- 自定义节点发布前需自测通过

4. 安全与合规底座
- 敏感信息脱敏
- 审计日志留痕

5. 规则配置与发布
- 规则创建/编辑/启停
- 规则版本化
- 按机构/系统/角色灰度生效

6. 运行数据采集能力
- 智能提示运行数据（触发量、命中量、采纳量）
- 智能作业运行数据（查询量、注入成功率、失败原因分布）

## 1.2 P1（V1.1 紧随其后）
1. 提示策略增强
- 多规则冲突治理可视化
- 提示去重与疲劳控制

2. 智能作业增强
- 注入前差异预览
- 低置信度数据人工确认流程

3. 运营增强
- A/B 策略效果对比
- 机构与岗位分层分析报表

4. 可用性增强
- 离线缓存策略优化
- 内部系统数据源容灾切换策略

## 2. 阶段目标（不含时间）

- 冻结 P0 范围与验收口径
- 完成首批业务系统接入清单
- 产出 JSSDK 接入规范 v1

2. 阶段 2（JSSDK 主链路）
- 完成初始化、上下文采集、请求链路
- 打通日志回传与基础埋点

3. 阶段 3（智能提示能力）
- 落地监管传达、风险提醒、新员工作业指引三类触发
- 完成提示优先级与展示规则
- 完成关键页面联调（首批）

4. 阶段 4（智能作业能力）
- 接入首批内部系统数据源
- 完成字段映射与注入策略

- 完成场景编排执行引擎与首批节点落地

5. 阶段 5（规则配置与灰度）
- 完成规则管理后台核心操作
- 完成版本化与灰度发布

6. 阶段 6（联调与压测）
- 完成端到端联调

- 输出运营手册与应急预案
- 满足上线门禁后执行全量发布

## 3. 上线门禁（Go/No-Go）
1. 提示触发端到端 P95 <= 800ms。
2. 智能作业端到端 P95 <= 1500ms。
3. 关键页面注入成功率 >= 98%。
4. 生产高风险提醒漏报率 <= 1%。
5. 审计日志完整率 = 100%。
6. P0 阻断缺陷清零。

## 4. 角色分工建议
1. 产品
- 规则口径确认、范围管控、验收把关。

2. 前端/JSSDK
- 注入框架、提示渲染、字段注入、浏览器兼容。

3. 后端平台
- 规则引擎、数据聚合、灰度发布、日志与统计。

4. 测试
- 关键路径自动化、风险场景回归、性能验证。

5. 业务与合规
- 提示内容审核、风险规则确认、试点评估。

## 5. 当前关键风险
1. 内部系统数据源 SLA 不稳定影响注入时效。
2. 业务系统页面结构差异导致适配成本偏高。
3. 风险提醒仅做提示且不改变现有流程，需重点关注提示疲劳与采纳率。

## 6. 确认记录
1. P0-1（JSSDK 基础运行框架）已确认。
2. P0-2 调整为“统一提示引擎”，监管传达/风险提醒/新员工作业指引仅作为业务分类。
3. P0-3（智能作业核心能力）已确认。
4. P0-4 调整为仅保留“敏感信息脱敏、审计日志留痕”。
5. P0-5（规则配置与发布）已确认。
6. P0-6 调整为仅采集“智能提示、智能作业”运行数据。
7. 试点范围确认为开阳业务系统；业务入口按“专区 -> 业务菜单 -> 业务页面”组织。
8. 智能提示与智能作业启用粒度确认为“业务菜单级”，由管理员控制启停。
9. 数据源范围确认为“仅内部系统数据源”。
10. 默认注入策略按“业务菜单级配置”执行，不设全局统一默认。
11. 新员工标记方式确认为“人工标记”，且不改变现有流程。
12. 智能提示生效策略确认为“IP + 人员 + 机构”混合生效。
13. 风险提醒确认为“不改变现有流程”，仅做提示与记录。
14. 智能提示条件运算符确认为“常见运算符全支持”。
15. DOM 定位方式确认为“XPath”。
16. 触发链路确认为“外部 SDK 监听 DOM -> 通知智能提示 SDK -> 执行规则 -> 满足即弹窗”。
17. 智能作业触发支持灵活配置，并复用智能提示规则能力；提示命中后可按配置执行智能作业。
18. 提示命中后的联动作业触发时机确认为“提示渲染后执行”。
19. 多条提示同轮命中时，确认为“合并到同一窗口展示”。
20. 多个联动作业确认为“按智能提示触发顺序执行”。
21. DOM 高频变化场景确认为“防抖处理，避免重复弹窗和重复联动”。
22. 智能作业注入完成后确认为“不再进行后续处理”。
23. 智能作业新增预览模式：填充前可预览并修改，确认后写入；取消则不注入。
24. 智能作业支持页面悬浮按钮触发，用于重复触发已绑定作业。
25. 智能作业触发规则确认为“完全由智能提示控制”，不提供独立触发。
26. 智能提示条件表达式扩展为“左值/右值支持固定值、页面元素、接口返回值”。
27. 接口确认为配置管理子项，需用户提前配置定义并发布后方可被规则引用。
28. 接口子项配置模型确认为字段化管理，状态流转为 `DRAFT`/`PUBLISHED`/`OFFLINE`。
29. 接口版本策略当前不启用，但保留扩展字段与后续能力演进空间。
30. 被已发布规则引用的接口不允许直接下线，需先解除引用。
31. 接口发布前校验确认包含：连通性、鉴权有效性、`response_path` 可解析性。
32. 接口调用参数确认：`timeout_ms`、`retry_count` 由平台统一控制（默认 1000/1），不对用户开放配置。
33. 接口发布/下线权限确认：仅接口 `owner` 与平台管理员可执行。
34. 前端确认：接口配置页面不展示 `timeout_ms`、`retry_count`，用户不可编辑。
35. 接口参数支持部分依赖页面取值：调用前按映射先取页面值再组装请求。
36. 参数映射项确认新增 `test_value`，用于测试模式模拟取值。
37. 参数类型默认按字符串处理（未配置 `value_type` 时）。
38. 智能提示条件表达式左值与右值支持独立预处理器链并顺序执行（首批：提取前几位、数字转换）。
39. 日志策略确认：参数与调用日志全量落库，查询展示按敏感规则脱敏。
40. 接口参数映射层不使用预处理器链，保持参数组装逻辑简化。
41. 预处理器实现方式确认为 JavaScript，且在平台沙箱执行。
42. 首批预处理器确认为“提取前几位”“数字转换”。
43. 预处理器扩展权限确认为系统管理员（可新增/启停）。
44. 智能作业支持场景编排，编排由节点序列组成，首批节点包含页面取值、页面赋值、接口调用、JavaScript 脚本。
45. 编排节点支持扩展，自定义节点由系统管理员发布与启停。
46. 编排关系确认为“一作业一编排”，触发时执行该编排全链路。
47. 编排失败策略确认当前且唯一为 `STOP`，前端不开放“继续执行”配置。
48. 悬浮按钮重复触发确认：每次点击创建新编排实例，不复用上次实例上下文。
49. `js_script` 节点口径确认：当前不限制直接改页面对象。
50. 自定义节点发布口径确认为“自测通过后发布”。
51. 预览时机确认：编排全部计算完成后统一预览，确认后一次性写入页面。
